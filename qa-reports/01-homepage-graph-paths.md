> 測試裝置：Desktop 1440x900 / Mobile 375x812
> 瀏覽器：Chromium (Playwright headless)

## 一、首頁 (`/`)

### 截圖分析

**Desktop 首屏 (Above the Fold)**：
- 頁面頂部有 "Blockchain Wiki" 品牌名稱，搜尋列，GitHub 連結
- H1 標題 "Ethereum 學習平台" 清楚可見，副標題 "從密碼學到共識機制，深入理解 Ethereum 協議"
- 有一個藍色 CTA 按鈕 "開始學習 ->"
- 下半部分可以看到 "知識圖譜" 區塊的分類按鈕（密碼學、資料結構、帳戶與交易、交易流程、區塊與共識、進階主題）

**Desktop 完整頁面**：
- 知識圖譜區域是一個巨大的黑色空白區塊（渲染失敗）
- 下方有 6 個學習路徑卡片，全部顯示 0% 進度
- 最底部有 "內容概覽" 文字列表
- 頁面結構：Hero -> 知識圖譜 -> 學習路徑 -> 內容概覽

**Mobile**：
- 版面適配良好，沒有水平溢出
- CTA 按鈕置中顯示
- 分類按鈕自動換行（3+3 排列）
- 知識圖譜同樣是黑色空白
- 路徑卡片垂直堆疊，寬度適當

### 內容分析

- 頁面標題：`Ethereum 學習平台 | Blockchain Wiki`
- Meta description：`從密碼學到共識機制，深入理解 Ethereum 協議`
- 語言設定：`zh-TW`，字型：`Noto Sans TC`
- 統計顯示：49 篇筆記 / 298 個連結

### 問題列表

| # | 嚴重度 | 問題描述 |
|---|--------|----------|
| 1 | **CRITICAL** | 知識圖譜 React 元件 hydration 失敗，整個圖譜區域顯示為空白黑色畫布。Console 錯誤：`[astro-island] Error hydrating /src/components/KnowledgeGraph.tsx TypeError: Failed to fetch dynamically imported module` 及 `504 (Outdated Optimize Dep)` |
| 2 | **HIGH** | 截圖中所有中文字元顯示為方塊 (tofu characters)。這是 headless 瀏覽器缺少字型的問題，但應確認實際瀏覽器中字型正常載入。Noto Sans TC 字型需確認已正確安裝或透過 CDN 載入 |
| 3 | **HIGH** | "開始學習" CTA 按鈕直接跳轉到 `/ethereum/transaction-lifecycle/transaction-lifecycle/`（交易生命週期），對初學者來說跳得太深。應該連到概述頁面或第一條學習路徑（密碼學基礎） |
| 4 | **MEDIUM** | 沒有 `<nav>` 導航元素。頂部只有品牌名稱 + 搜尋 + GitHub，缺少主要導航連結（知識圖譜、學習路徑等入口） |
| 5 | **MEDIUM** | 學習路徑卡片不可點擊展開。儘管有向下箭頭圖示（chevron），點擊卡片沒有反應。使用者期望能點開看到路徑中的具體文章列表 |
| 6 | **MEDIUM** | "內容概覽" 區塊只是純文字列表，沒有連結。使用者無法從這裡直接跳轉到對應主題 |
| 7 | **LOW** | Starlight 框架的 "跳转到内容" 連結使用簡體中文，與網站的繁體中文不一致 |
| 8 | **LOW** | 搜尋功能的文字也有簡體/繁體混用：搜尋框內顯示 "搜索"（簡體），主題選擇器顯示 "选择主题"、"深色"、"浅色"、"自动" |
| 9 | **LOW** | 頁面 anchor link `#學習路徑` 和 `#內容概覽` 在 Playwright 的 `page.goto()` 中返回 `NO_RESPONSE`，不過這可能是測試工具的問題而非真實 bug |

## 二、知識圖譜 (`/graph/`)

### 截圖分析

**Desktop**：
- H1 標題 "知識圖譜"
- 有 6 個分類篩選按鈕（密碼學、資料結構、帳戶與交易、交易流程、區塊與共識、進階主題）
- 圖譜本身是完全黑色的空白畫布（Canvas + SVG 元素存在，但未渲染任何節點）
- 下方顯示 "49 篇筆記 / 298 個連結"
- 有使用說明文字："點擊節點跳轉到對應筆記，將滑鼠移到節點上可查看其關聯..."
- 左側側邊欄完整顯示所有內容分類和文章列表
- 底部有上一頁/下一頁導航

**Mobile**：
- 側邊欄收起為漢堡選單
- 圖譜區域佔滿寬度但仍然全黑
- 使用說明文字正常換行顯示

### 互動測試

| 測試項目 | 結果 |
|----------|------|
| 點擊圖譜節點 | 失敗 - 元素不可見（因為根本沒有渲染） |
| 滑鼠滾輪縮放 | 技術上成功（無報錯），但畫布仍為空白 |
| 滑鼠拖曳平移 | 技術上成功（無報錯），但畫布仍為空白 |
| 分類篩選按鈕 | 全部超時失敗（hydration 錯誤導致 React 元件未掛載） |

### 問題列表

| # | 嚴重度 | 問題描述 |
|---|--------|----------|
| 1 | **CRITICAL** | 知識圖譜完全無法渲染。Console 錯誤同首頁：`[astro-island] Error hydrating KnowledgeGraph.tsx`。這是此頁面的核心功能，完全失效 |
| 2 | **CRITICAL** | 分類篩選按鈕全部無法點擊（因 React 元件未 hydrate）。整個互動式功能完全無法使用 |
| 3 | **HIGH** | 沒有圖例（legend）。雖然按鈕使用了不同顏色的圓點標示分類，但圖譜本身沒有獨立的圖例面板 |
| 4 | **HIGH** | 沒有縮放控制 UI。雖然滑鼠滾輪可以操作，但沒有視覺化的 +/- 按鈕，觸控裝置使用者無法操作 |
| 5 | **MEDIUM** | 頁面底部的上一頁連到 "Transaction Lifecycle"，下一頁連到 "學習路徑"。從知識圖譜的角度來看，這個導航順序對使用者來說不直覺 |

## 三、學習路徑 (`/paths/`)

### 截圖分析

**Desktop**：
- H1 標題 "學習路徑"
- 說明文字："選擇一條路徑，按步驟學習。進度自動儲存在瀏覽器中。"
- 6 張路徑卡片垂直排列，每張包含：
  - 左側彩色圓點（分類色）
  - 路徑名稱（粗體）
  - 路徑描述
  - 右側進度百分比（0%）
  - 下拉箭頭
- 有左側邊欄導航
- 底部有上一頁/下一頁導航

**Mobile**：
- 卡片全寬顯示，堆疊良好
- 卡片內容不溢出
- 側邊欄收起

### 內容分析

6 條學習路徑：
1. **密碼學基礎路徑** - 從雜湊函數到數位簽章，理解 Ethereum 的密碼學基石
2. **帳戶與交易路徑** - 帳戶模型、地址體系與交易費用機制
3. **交易流程路徑** - 一筆交易從發起到上鏈的完整生命週期
4. **資料結構路徑** - 從編碼格式到各種 Trie 結構的完整數據流
5. **區塊與共識路徑** - Proof of Stake 共識機制與區塊結構全貌
6. **進階主題路徑** - Danksharding、零知識證明與下一代狀態結構

側邊欄導航結構使用 `<details>` 元素分類（Ethereum > 密碼學 > SHA-256, Keccak-256...），結構完整。

### 連結驗證

所有 70 個內部連結全部返回 200 OK（除了 2 個 anchor link 在測試工具中返回 NO_RESPONSE）。沒有發現真正的壞連結。

### 問題列表

| # | 嚴重度 | 問題描述 |
|---|--------|----------|
| 1 | **HIGH** | 路徑卡片沒有明確的 "開始" 按鈕或連結。卡片視覺上暗示可點擊（有 chevron），但沒有 `<a>` 標籤。使用者不知道怎麼開始學習 |
| 2 | **HIGH** | 路徑卡片展開後（如果能展開的話）應顯示具體文章清單。目前的結構使用 `<details>` 元素但似乎存在於側邊欄而非主要內容區域 |
| 3 | **MEDIUM** | 路徑沒有推薦起始路徑標記。對初學者來說，不知道該從哪條開始。建議為 "密碼學基礎路徑" 加上 "推薦起始" 標記 |
| 4 | **MEDIUM** | 進度追蹤（0%）存在但機制不明。使用者不清楚如何讓進度增加（是閱讀完文章自動更新？還是需要手動標記？） |
| 5 | **LOW** | 路徑描述和對應的實際文章數量沒有顯示。例如 "密碼學基礎路徑" 沒有標示包含幾篇文章 |
| 6 | **LOW** | 頁面標題沒有 H2/H3 子標題。所有路徑是平面列表，沒有難度分級或分組 |

## 六、優先改善建議

### P0 (Must Fix - 上線前必須修復)

1. **修復 KnowledgeGraph.tsx hydration 錯誤** - 整個知識圖譜功能完全失效。檢查動態 import 路徑和 Vite optimize dep 設定。可能需要重啟 dev server 或清除 `.astro` 快取
2. **修復 "開始學習" CTA 連結目標** - 應連到 `/paths/` 或第一條學習路徑的第一篇文章（如 `/ethereum/cryptography/sha-256/`），而非直接跳到交易生命週期

### P1 (Should Fix - 高度建議修復)

3. **增加頂部導航** - 在 header 中加入主要頁面連結：首頁、知識圖譜、學習路徑
4. **讓學習路徑卡片可點擊展開** - 展開後顯示該路徑包含的文章列表，每篇文章可直接點擊進入
5. **為初學者標記推薦起始路徑** - 在 "密碼學基礎路徑" 加上 "建議起點" 的標記

### P2 (Nice to Have)

6. **統一簡繁體中文** - 將 Starlight UI 的 "搜索"、"选择主题" 等改為繁體
7. **在路徑卡片中顯示文章數量** - 如 "密碼學基礎路徑 (8 篇)"
8. **為知識圖譜增加縮放控制 UI** - 方便觸控裝置使用
9. **讓 "內容概覽" 區塊的條目可點擊** - 加入超連結到對應分類頁面
10. **說明進度追蹤機制** - 在學習路徑頁面簡短說明進度如何計算
